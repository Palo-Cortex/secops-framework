## 3.7.1

##### JOB - Auto Triage V3
New scheduled JOB that automatically closes low-signal cases that have exceeded
the configured age threshold without analyst attention. Cases are excluded if
starred, if `manual_score` is set, or if `aggregated_score` exceeds the
configured threshold. Enable and disable via the XSIAM Jobs UI. Thresholds
are controlled via `SOCOptimizationConfig_V3` (`TriageWindowHours`,
`TriageScoreThreshold`).

##### SOCAutoTriageScoreFilter (new script)
Filters the `get_incidents` API response by `aggregated_score` and `manual_score`.
Returns two arrays: `AutoTriage.filtered_incidents` (eligible for close) and
`AutoTriage.skipped_incidents` (above threshold or analyst-touched). Accepts
float threshold values for sub-hour testing (e.g. `0.1`).

##### value_tags.json
Added `auto_triage_closed` and `auto_triage_skipped` entries (Category: `triage`,
VD3/MTTA) to enable auto-triage case counts in the Value Metrics Dashboard.
