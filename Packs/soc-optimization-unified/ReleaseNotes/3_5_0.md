## [3.5.0] - March 2026

### IncidentFields (new)

Added 12 generic email incident fields to `soc-optimization-unified`. These fields replace
vendor-specific `proofpointtap*` fields with a product-agnostic namespace, enabling any
email security source (TAP, Defender for O365, Mimecast, etc.) to write to a single
consistent contract consumed by `Foundation_-_Normalize_Email_V3` and the Email NIST IR
lifecycle.

All fields use the `socfw` prefix to prevent collision with Marketplace content. All fields
have `associatedToAll: true` making them available across all alert types.

| Field | cliName | Type | Purpose |
|---|---|---|---|
| Email Delivery Action | `socfwemaildeliveryaction` | Single select | Routing gate: delivered / blocked / click_permitted / click_blocked |
| Email Direction | `socfwemaildirection` | Single select | Containment scope: inbound / outbound / internal |
| Email Threat URL | `socfwemailthreaturl` | Short text | soc-block-indicator target, IOC grouping |
| Email Threat Type | `socfwemailthreattype` | Single select | Analysis branch: url / attachment / domain |
| Email Threat Status | `socfwemailthreatstatus` | Single select | Urgency gate: active / cleared / malicious |
| Email Threat ID | `socfwemailthreatid` | Short text | Vendor threat tracking reference |
| Email Classification | `socfwemailclassification` | Single select | phish / malware / spam / impostor |
| Email Phish Score | `socfwemailphishscore` | Number | 0–100, feeds Analysis.confidence |
| Email Malware Score | `socfwemailmalwarescore` | Number | 0–100, gates soc-detonate-file |
| Email Campaign ID | `socfwemailcampaignid` | Short text | Primary campaign grouping key |
| Email Click IP | `socfwemailclickip` | Short text | Lateral risk detection, click-source grouping |
| Email Click Time | `socfwemailclicktime` | Short text | Click event timestamp (ISO 8601) |

#### Breaking change
The `proofpointtap*` custom fields are removed from this pack. The TAP correlation rule
in `SocFrameworkProofPointTap` must be updated to write to `socfw*` fields before
deploying 3.5.0. See `feature/email-normalization-fields` PR for the coordinated rule update.
