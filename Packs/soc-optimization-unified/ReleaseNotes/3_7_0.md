## 3.7.0

#### SOC Framework Unified

##### JOB - Triage Alerts V3
- Added score-gated auto-close. Incidents with an `aggregated_score` at or below `TriageScoreThreshold` are now the only candidates for auto-close. Incidents scored above the threshold are excluded from the triage window close regardless of starred status, preserving analyst visibility on cases XSIAM has already ranked as significant.

##### Foundation - Dedup V3
- Added score-based dedup bypass. When a parent incident's `predicted_score` meets or exceeds `DedupScoreThreshold`, dedup is skipped and the alert surfaces as an independent case. Prevents escalating signals from being absorbed into an already high-risk active case.

##### SOCOptimizationConfig V3
- Added `TriageScoreThreshold` (default: `30`) to `Triage Incidents JOB` fields. Controls the minimum `aggregated_score` below which unstarred, aged-out incidents are eligible for auto-close.
- Added `DedupScoreThreshold` (default: `70`) to `Dedup` fields. Controls the minimum parent `predicted_score` at which dedup is bypassed for incoming alerts.
