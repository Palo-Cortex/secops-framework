## 1.1.0

#### Correlation Rule — SOC Trend Micro Vision One V3

- Fixed `alert_category` from `OTHER` to `User Defined` — resolves 101704 installation failure on upload
- Fixed `fromversion` to `6.10.0` — `8.0.0` caused HTTP 500 on upload
- Restored `rule_id: 0` — required for platform runtime resolution; removing it caused silent misfires
- Corrected MITRE `alert_fields` key names to `mitretacticid`, `mitretacticname`, `mitretechniqueid`, `mitretechniquename` — previous names (`mitreid`, `mitrename`) silently failed to populate MITRE fields on alerts
- Set `alert_type: null`, `crontab: null`, `search_window: null`, `simple_schedule: null`, `timezone: null` explicitly — omitting optional fields caused HTTP 500 on upload
- Populated `mitre_defs` with full tactic/technique map (was empty `{}`)
- Added `suppression_duration: 1 hours` with suppression on `id` field — prevents duplicate alerts per workbench event
- Added `investigation_query_link` with dataset-scoped XQL
- Added `user_defined_category: tactic`
- Added MITRE tactic routing fields: `mitretacticid` and `mitretacticname` mapped to resolved tactic name and TA-number
- Expanded MITRE tactic arrays in XQL to cover all 14 ATT&CK tactics
- Fixed `xdm.source.host.hostname` in modeling rule — was using `entity_value.guid` (UUID) instead of `entity_value.name`; UUID in hostname field breaks UEBA and BIOC entity correlation
- Expanded `xdm.alert.mitre_tactics` enum maps in all three modeling rule filter blocks (workbench, search_detection, observed_attack_technique) — sparse maps were silently dropping unmapped tactics from analytics
- Cleaned `_schema.json` — removed intermediate XQL variable names that leaked into schema; added raw dataset fields `created_date_time`, `updated_date_time`, `model_id`, `schema_version`
- Removed corrupt empty-key dependency from `pack_metadata.json` — was causing 101704 on pack installation
