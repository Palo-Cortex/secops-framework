## [1.4.7] - April 2026

### Overview

Fixes layout script installation failures that caused NIST IR case layout sections
(Analysis, Containment, Eradication, Recovery, Endpoint Status) to error on render.
Adds post-installation documentation and a rewritten README.

---

### Bug Fixes

#### All layout scripts — bundle installer silently skipped (Critical)

All seven layout and response scripts were missing the `contentitemexportablefields`
block required by the pack bundle installer. The SDK upload path (`demisto-sdk upload`)
does not enforce this field, so scripts installed correctly in dev but were silently
dropped on every bundle install. Affected scripts:

- `SOCFramework_displayAnalysis`
- `SOCFramework_displayContainment`
- `SOCFramework_displayEradication`
- `SOCFramework_displayRecovery`
- `SOCFramework_displayEndpointStatus`
- `SOCFramework_ManualIsolateEndpoint`
- `SOCFramework_ManualDeisolateEndpoint`

Fix: `contentitemexportablefields` block added to all seven script YAMLs.

#### All layout scripts — `hidden: true` rendered scripts disabled in layout editor

All seven scripts had `hidden: true`, which caused the XSIAM layout editor to mark
them as disabled and prevented the layout engine from resolving them at runtime.
Error observed: `unable to find script [SOCFramework_displayContainment] to execute (2603)`.

Fix: `hidden: false` set on all seven scripts.

### Added

#### `POST_CONFIG_README.md`

New post-installation guide covering the two tenant-level manual steps required after
pack install: Automation Rule (`SOC Framework NIST IR`, severity filter, EP playbook
assignment) and Layout Rule (`NIST IR`, `SOCFramework NIST IR Layout`). Includes
troubleshooting steps for the blank layout rule binding issue that occurs after
pack reinstalls.

#### `xsoar_config.json` — `post_config_docs` entry

Added `post_config_docs` block pointing to `POST_CONFIG_README.md` on GitHub,
consistent with the pattern used by other SOC Framework vendor packs.

---

### Changed

#### `README.md` — Complete rewrite

Previous README contained interleaved content from the `soc-framework-manager` pack.
Replaced with accurate documentation covering pack architecture, the four-layer playbook
model, full playbook inventory by phase and category, incident fields, scripts, Shadow
Mode behavior, Value Driver alignment, dependencies, and installation steps.
Post-installation steps moved to `POST_CONFIG_README.md`.
