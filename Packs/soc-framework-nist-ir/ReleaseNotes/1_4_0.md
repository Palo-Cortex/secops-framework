## [1.4.0] - March 2026

### Overview

This release introduces **signal-type-aware response** across all Endpoint phase playbooks.
The Endpoint category now differentiates between three signal types established during Signal
Characterization â€” `behavior_memory`, `process_execution`, and `file_malware` â€” and routes
Containment, Eradication, and Recovery actions accordingly. This prevents unnecessary host
isolation for memory/injection-based attacks, triggers credential remediation when credential
access tactics are detected, and validates process artifact cleanup on execution-based signals.

All new branches execute via `SOCCommandWrapper` and respect Shadow Mode. No existing paths
are removed or altered.

### Bug Fixes

#### SOC_EndPoint_Recovery_V3 â€” Dead end at task 18 (Critical)

Task 18 (Set Recovery Story) pointed to task 202 which does not exist in the playbook.
This caused Recovery to silently complete without executing any recovery actions (no
deisolation, no re-enable user, no story set) on every alert. Fixed: task 18 now routes
to task 500 (signal-type router), which routes to the correct recovery action path.

### Notes

- The `case_mitre_tactics` input in `SOC_EndPoint_Eradication_V3` reads from
  `parentIncidentFields.mitre_tactic_id_and_name`. This is a case-level field populated
  by XSIAM's alert grouping engine. If a case is not fully grouped at playbook execution
  time, this field may be empty and the credential access check will fall through to the
  `#default#` path (kill process without token revocation). This is safe â€” no action is
  skipped that would cause harm. A future improvement could read tactic IDs directly from
  the alert's own MITRE fields.
- All six Endpoint phase playbooks have been validated against the Turla Carbon scenario
  (138 CrowdStrike events: 71% `behavior_memory`, 29% `process_execution`). Expected
  execution in Shadow Mode: `soc-block-indicators` + `soc-revoke-tokens` for credential
  access users (bannik, Frieda, SYSTEM, evals_domain_admin) + `soc-kill-process` for all
  process execution signals + `soc-enable-user` on recovery.
