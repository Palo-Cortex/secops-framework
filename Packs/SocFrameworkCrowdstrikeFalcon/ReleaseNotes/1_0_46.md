## [1.0.46] - March 2026

### Changed

#### SOC CrowdStrike Falcon - Endpoint Alerts — Correlation rule rewrite

The CrowdStrike Falcon correlation rule was rewritten to improve alert naming, enrichment,
and suppression behavior for the SOC Framework NIST IR lifecycle.

**Alert naming** — `alert_name` now uses the SOC Framework `[Endpoint]` prefix pattern:

```
[Endpoint] {user_name} - {tactic}: {technique}
```

This aligns with the Email correlation rule's `[Email]` prefix, enabling cross-source
grouping in the XSIAM grouping engine via consistent `alert_name` format matching.

**Alert description** — enriched with host, user, and severity context:

```
{original_description} | Host: {hostname} | User: {user_name} | Severity: {severity_name}
```

**Suppression** — suppression is enabled on `composite_id` with a 1-hour window.
`composite_id` is the CrowdStrike-native unique detection identifier. Suppression on this
field prevents duplicate alerts from the same detection event while preserving visibility
across distinct detections for the same host and user.

**Replay compatibility** — the `replay_scenario.py` tooling randomizes `composite_id`
values per run (appends `_{run_id}`) to bypass the 1-hour suppression window during
testing without modifying the rule configuration.

**YAML encoding** — all non-ASCII characters (em-dashes, arrows, box-drawing) removed.
Non-ASCII characters in correlation rule YAML cause error 101704 on upload.
