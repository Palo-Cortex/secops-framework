## [1.1.1] - March 2026

### Changed

#### SOC CrowdStrike Falcon - Endpoint Alerts — Correlation rule consolidation

Replaced 15 per-tactic correlation rules (one per MITRE tactic) with a single consolidated
rule. The per-tactic split was a workaround for a previous XSIAM GUI limitation that no
longer applies.

**Alert category** is now dynamically set from the MITRE `tactic` field using
`user_defined_category: tactic`, replacing 15 hardcoded `alert_category` values. All
12 MITRE tactics are covered.

**Backwards compatibility** — five `alert_fields` mappings present in the 1.0.14
per-tactic rules were absent from the intermediate consolidated rule and are restored:

- `causality_actor_process_command_line`, `causality_actor_process_image_name`,
  `causality_actor_process_image_path` — CGO fields required for XSIAM issue grouping
- `deviceou` → `device_ou_arr` — OU array field (XQL `arraymap` alter also restored)
- `filehash` — generic hash field used by downstream threat intel lookups

**Schema fixes** — corrected `id`, `ruleid`, and `fromversion` fields that were
missing or set incorrectly, causing silent SDK validation gaps. Removed rogue
`rule_id: 0` that caused the SDK to resolve `id` as null.

**Deploy state** — ships with `is_enabled: false` to support side-by-side testing
alongside existing 1.0.14 rules. Enable this rule and disable the 1.0.14 per-tactic
rules to cut over. Roll back by reversing the toggle states.

#### SOC CrowdStrike Falcon Modeling Rule — Schema fix and new XDM mappings

**Fixed** — `fromversion` corrected from `8.3.1` to `6.10.0`. The incorrect version
caused error 101704 (pack installation failure) on upload.

**Fixed** — `local_process_id` was used in the XIF (`to_integer(local_process_id)`)
but was not declared in `schema.json`. Added to schema.

**Added** to `schema.json` — `composite_id`, `agent_id` (newly mapped source fields).

**Added** XDM mappings:

- `xdm.event.id` = `composite_id` — stable unique event identifier
- `xdm.source.host.device_id` = `device->device_id` — CrowdStrike AID
- `xdm.source.agent.identifier` = `agent_id` — Falcon sensor identifier

### Pack

**Pack ID changed** from `soc-crowdstrike-falcon` to `soc-crowdstrike-falcon`.

The previous pack remains installed under the old ID. Installing this pack does not
overwrite or remove layouts, scripts, or correlation rules from the previous version.
Both packs can coexist on the same tenant, enabling side-by-side testing before cutover.

**Layouts and Scripts removed.** `Layouts/` and `Scripts/` directories have been removed
from this pack. These belong in the SOC Framework core packs, not in vendor integration
enhancement packs. The correlation rule and modeling rule are the only content items.
